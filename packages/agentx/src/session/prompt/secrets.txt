Here are the exact instructions you MUST use when asking the user for API keys and secrets. Provide the direct URL and clear numbered steps.

Clerk:
"I need your Clerk Publishable Key and Secret Key for authentication."
URL: https://dashboard.clerk.com/
1. Go to the Clerk Dashboard and select your application.
2. Click "API Keys" in the left sidebar under "Developers".
3. Copy the "Publishable key" (starts with pk_) and "Secret key" (starts with sk_).

Supabase:
"I need your Supabase URL and Anon Key for the database."
URL: https://supabase.com/dashboard/
1. Go to your Supabase project dashboard.
2. Click the Settings gear icon in the left sidebar, then select "API".
3. Copy the "Project URL" and the "anon" public key.

Firebase:
"I need your Firebase config object."
URL: https://console.firebase.google.com/
1. Go to your Firebase Console and select your project.
2. Click the gear icon next to "Project Overview" and select "Project settings".
3. Scroll down to "Your apps", select the web app, and copy the `firebaseConfig` object (apiKey, authDomain, projectId, etc).

Appwrite:
"I need your Appwrite Endpoint and Project ID."
URL: https://cloud.appwrite.io/
1. Go to your Appwrite console and select your project.
2. Click "Settings" at the bottom of the left sidebar.
3. Copy the "API Endpoint" and "Project ID".

MongoDB Atlas:
"I need your MongoDB Atlas Connection String."
URL: https://cloud.mongodb.com/
1. Go to the MongoDB Atlas dashboard.
2. Click "Database" in the left sidebar, then click "Connect" on your cluster.
3. Choose "Drivers", select Node.js, and copy the connection string. Replace `<password>` with your database user password.

PlanetScale:
"I need your PlanetScale Connection String."
URL: https://app.planetscale.com/
1. Go to your PlanetScale dashboard and select your database.
2. Click "Connect", select "Node.js" as the language and "Prisma" or "Drizzle" as the framework if applicable.
3. Copy the `DATABASE_URL` string provided.

Stripe:
"I need your Stripe Secret Key for payments."
URL: https://dashboard.stripe.com/apikeys
1. Go to the Stripe Dashboard and click "Developers" -> "API keys".
2. Ensure you are in "Test mode" for development.
3. Reveal and copy the "Secret key" (starts with sk_test_).

Lemon Squeezy:
"I need your Lemon Squeezy API Key for payments."
URL: https://app.lemonsqueezy.com/settings/api
1. Go to Settings -> API in your Lemon Squeezy dashboard.
2. Click "Create API key", give it a name, and copy the key.

Resend:
"I need your Resend API Key for email."
URL: https://resend.com/api-keys
1. Go to the Resend dashboard and click "API Keys" in the sidebar.
2. Click "Create API Key", give it "Full Access", and copy the key (starts with re_).

SendGrid:
"I need your SendGrid API Key for email."
URL: https://app.sendgrid.com/settings/api_keys
1. Go to the SendGrid dashboard, expand "Settings", and click "API Keys".
2. Click "Create API Key", choose "Restricted Access" (Mail Send), and copy the key (starts with SG.).

Postmark:
"I need your Postmark Server API Token."
URL: https://account.postmarkapp.com/servers
1. Select your Server in the Postmark dashboard.
2. Click "API Tokens" and copy the "Server API token".

Vercel:
"I need your Vercel Access Token for deployment."
URL: https://vercel.com/account/tokens
1. Go to your Account Settings -> Tokens in Vercel.
2. Click "Create Token", give it a name, and copy the token.

Netlify:
"I need your Netlify Personal Access Token for deployment."
URL: https://app.netlify.com/user/applications#personal-access-tokens
1. Go to User Settings -> Applications.
2. Under "Personal access tokens", click "New access token" and copy it.

Railway:
"I need your Railway API Token."
URL: https://railway.app/account/tokens
1. Go to your Account Settings -> Tokens.
2. Create a new token and copy it.

Fly.io:
"I need your Fly.io Access Token."
URL: https://fly.io/dashboard/personal/tokens
1. Go to your Fly.io Dashboard -> Account -> Access Tokens.
2. Create a new token and copy it.

Cloudflare Pages / R2:
"I need your Cloudflare API Token and Account ID."
URL: https://dash.cloudflare.com/profile/api-tokens
1. Go to My Profile -> API Tokens.
2. Click "Create Token" (use the Edit Cloudflare Workers/Pages template).
3. Your Account ID can be found in the URL or sidebar of your zone dashboard.

AWS S3:
"I need your AWS Access Key ID and Secret Access Key."
URL: https://console.aws.amazon.com/iam/home#/security_credentials
1. Go to the IAM Console -> Users, select your user.
2. Click "Security credentials", scroll to "Access keys", and click "Create access key".

Uploadthing:
"I need your Uploadthing Secret."
URL: https://uploadthing.com/dashboard
1. Go to your Uploadthing dashboard and select your app.
2. Click "API Keys" and copy the "Secret".

GitHub Personal Access Token:
"I need your GitHub Personal Access Token (Classic) to create and push to the repo."
URL: https://github.com/settings/tokens/new
1. Go to GitHub Settings -> Developer Settings -> Personal access tokens -> Tokens (classic).
2. Click "Generate new token (classic)".
3. Give it a note, set expiration, and check the "repo" and "workflow" scopes.
4. Click Generate and copy the token (starts with ghp_).

PostHog:
"I need your PostHog Project API Key and Host URL."
URL: https://us.posthog.com/project/settings
1. Go to your Project Settings in PostHog.
2. Under "Project Variables", copy the "Project API Key" and "Instance Address" (Host).

Sentry:
"I need your Sentry DSN."
URL: https://sentry.io/
1. Go to your Sentry dashboard and select your project.
2. Click "Settings" -> "Client Keys (DSN)".
3. Copy the DSN string.
